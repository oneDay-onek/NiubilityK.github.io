Js的运行机制

一、Js为单线程应用

js的用途决定了它是单线程。Js作为一个浏览器脚本语言，主要对用户交互和处理页面DOM负责。而如果Js是个多线程语言，将会造成很复杂的影响，在一篇博文中看到的原话是会有很复杂的同步问题，一点就懂。比如有两个线程，其中一个线程在修改一个DOM元素，另一个线程要删除该DOM元素，浏览器此时该以哪个线程为准？

因此，为了避免产生逻辑上的复杂性，所以Js自诞生以来就是单线程，也是该语言核心的特征。

为了利用多核cpu的计算能力，HTML5提出Web Worker标准，允许js脚本创建多个线程，但是子线程完全受主线程控制，且不得操作DOM，所有新标准并没有改变js单线程的本质。 想了解web worker可移步至廖雪峰老师的文章《web worker使用教程》

二、任务队列

单线程意味着任务只能一件一件去执行，如果遇到一个耗时较长的任务，排在后面的任务只能等待其完成再执行。由此出现了同步任务和异步任务，同步任务是指主线程上的任务只有当前一个任务执行完毕，后一个任务才会执行；异步任务是指不进入主线程而进入任务队列的任务，当异步任务结束后会将一个事件放进异步的队列。

当主线程里的同步任务都执行完后，会去异步的任务队列中按顺序执行异步任务添加的事件，也就是回调函数。如此反复，便形成了事件循环。（进入任务队列的都是回调函数）

补充点：异步任务也分为task（也叫macrotask，宏任务），和microtask（微任务）两种。

  1. 一个线程中，事件循环是唯一的，但是任务队列可以拥有多个。
  2. 任务队列又分为macro-task（宏任务）与micro-task（微任务），在最新标准中，它们被分别称为task与jobs。
  3. macro-task大概包括：script(整体代码)、setTimeout、setInterval、setImmediate(一次Event Loop循环之后调用)、I/O、UI rendering。
  4. micro-task大概包括: process.nextTick、Promise(promise主体函数为同步，.then()的回调才是异步任务)、MutationObserver(html5新特性，监听浏览器渲染或者重新渲染时调用内部回调)。
  5. 在当前的微任务没有执行完成时，是不会执行下一个宏任务的。可以理解为微任务优先。

三、事件和回调函数

  • “任务队列”是一个事件的队列，IO设备完成一项任务，就在”任务队列”中添加一个事件，表示相关的异步任务可以进入”执行栈”了。主线程读取”任务队列”，就是读取里面有哪些事件。
  • “任务队列”中的事件，除了IO设备的事件以外，还包括一些用户产生的事件（比如鼠标点击、页面滚动等等）。只要指定过回调函数，这些事件发生时就会进入”任务队列”，等待主线程读取。
  • 所谓”回调函数”（callback），就是那些会被主线程挂起来的代码。异步任务必须指定回调函数，当主线程开始执行异步任务，就是执行对应的回调函数。
  • “任务队列”是一个先进先出的数据结构，排在前面的事件，优先被主线程读取。主线程的读取过程基本上是自动的，只要执行栈一清空，”任务队列”上第一位的事件就自动进入主线程
